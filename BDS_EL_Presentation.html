Background

The Problem: Privacy vs Security

Network Intrusion Detection (IDS)

Companies monitor every network connection: who connected, how many bytes were sent, how long it lasted, which service was used. A machine learning model learns to classify connections as Normal or Attack.

The Privacy Problem

That data reveals sensitive activities of real people. If someone trains a model and shares it, an attacker can run Membership Inference Attacks — guessing whether a specific person's data was used in training.

Without DP: MIA success rate approaches 80–90% for outliers.

What is Differential Privacy?

Add carefully calibrated random noise so that removing or changing any single person's record doesn't noticeably change the output.

Pr[M(D) ∈ S] ≤ eε · Pr[M(D') ∈ S]

D and D' differ by one person. ε (epsilon) = privacy budget. Smaller ε = more private = more noise.

Core Concept

Input Perturbation — The Standard Approach

How It Works

  • 1
    Compute Sensitivity For each feature, calculate how much one person can change it: GS = (max − min) / n
  • 2
    Add Laplace Noise Noise = Laplace(0, GS/ε) added to every feature value in the dataset
  • 3
    Train on Noisy Data Normal ML model trains on the noisy version. Never sees real values.
Real: src_bytes = 45,000   →   Noisy: 47,832
Real: duration = 2.3s   →   Noisy: 1.1s
Real: count = 84   →   Noisy: 91

Why It FAILS Here

Total budget = ε = 1.0, split across 37 features:

ε_per_feature = 1.0 / 37 ≈ 0.027

ε = 0.027 requires enormous noise. The signal is completely drowned out. The model can only predict the majority class.

The Result

No DP (Baseline) 94.77%
Input Perturbation (ε=1.0) 53.3%

53.3% = predicting "Normal" for everything. Worse than a coin flip on attack detection. DP destroyed the model.

Solution

The Fix — DP-SGD

How DP-SGD Works

  • 1
    Train normally with mini-batches Use real data, standard neural network, subsampled batches
  • 2
    Clip each gradient For each training example, clip its gradient vector to norm ≤ C. Limits one person's influence.
  • 3
    Add Gaussian noise to gradient sum Noise = N(0, σ²C²I). One noise injection per training step.
  • 4
    Update model, track ε spent PRV accountant tracks exact privacy cost across all steps.

Key insight: One ε budget for the entire gradient — not split across 37 features. No budget collapse.

Novel Contribution #1

Clipped Sensitivity

The Problem with Standard GS

NSL-KDD feature src_bytes has max value = 1.3 billion.

GS = (max − min) / n = 1,300,000,000 / 125,974 ≈ 10,399

Laplace noise with this sensitivity is huge. The feature is useless.

1.3 billion is ONE extreme outlier. 99% of data has src_bytes < 26,000.

Our Fix — Clip at 99th Percentile

GS_clip = (τ − min) / n
τ = 99th percentile ≈ 26,000
GS_clip ≈ 0.203

Reduction: 10,399 → 0.203 = 51,000× less noise.

Formal Proof (We Wrote This)

LEMMA 1 — DP Validity

After clipping x_i to [min, τ], adjacent databases differ by at most (τ−min)/n in mean. Therefore Laplace(0, GS_clip/ε) is ε-DP. Clipping does NOT break privacy. □

LEMMA 2 — Bias Bound
|clipped_mean − true_mean| ≤ frac_above × (hi − τ)

Tells you exactly how wrong you can be. Controllable and provable.

Novel Contribution #2

Importance-Weighted Budget Allocation

Standard Approach (Broken)

ε_j = ε_total / d
Each of 37 features gets 0.027

Treats every feature equally — even useless ones get the same budget as critical attack indicators.

Our Approach

ε_j = ε_total × w_j
where Σ w_j = 1

Weight proportional to feature importance. Important features get MORE budget → LESS noise → BETTER signal.

Formal guarantee: By Basic Composition Theorem (Dwork & Roth 2014), since Σw_j=1, the joint mechanism is still exactly ε_total-DP.

Three Weighting Strategies

MI-Weighted
w_j ∝ Mutual Information with attack label
Recommended
Variance-Weighted
w_j ∝ Var(X_j) — high-variance features carry more signal
SNR-Heuristic
w_j ∝ std(X_j)/GS_j — balances signal vs sensitivity cost
Uniform (Baseline)
w_j = 1/d — standard approach, equal budget

Implemented in weighted_budget.py → allocate_budget() and compute_feature_importance()

Novel Contribution #3

PRV Accountant — Tighter Privacy Tracking

The Problem with Basic Composition

If you run k training steps, each costing ε₀:

Basic: ε_total = k × ε₀
At k=100, ε₀=0.5 → ε_total = 50.0

Way too pessimistic. Privacy budget "runs out" quickly and you can't train for long.

PRV — Privacy Loss Random Variable

Instead of tracking worst-case ε, track the actual distribution of privacy loss using Fourier convolution (FFT). Much tighter bounds.

f̃_{k} = f̃_{1} * f̃_{1} * ... * f̃_{1} (k convolutions)
via FFT in O(m log m) time

Comparison at k Queries (ε₀ = 0.5 per query)

Queries (k) Basic RDP PRV (Ours) Saving
10 5.000 5.074 3.667 28% tighter
20 10.000 9.850 5.656 43% tighter
50 25.000 19.318 9.928 49% tighter
100 50.000 31.543 15.335 51% tighter

At k=100, PRV allows 2× more training steps for the same privacy budget compared to RDP. More training = better accuracy.

Implemented in prv_accountant.py using FFT convolution of Privacy Loss Random Variables.

Summary

Key Takeaways

🚨
Finding #1 — Standard DP is Broken for Network IDS
Input perturbation across 37 features collapses to 53.3% accuracy at ALL epsilon values. The privacy budget is split too thin. This is not fixable by increasing ε.
✅
Finding #2 — DP-SGD Fully Solves It
Gradient-level DP bypasses feature-splitting. 94.70% accuracy at ε=1.0 on NSL-KDD (vs 94.77% non-private). Only 0.07 percentage point cost for full privacy.
✂️
Novel #1 — Clipped Sensitivity Reduces Noise by Up to 592 Million×
Clip at 99th percentile before computing sensitivity. Formally proven ε-DP (Lemma 1). Bias provably bounded (Lemma 2). Critical for heavy-tailed network data.
⚖️
Novel #2 + #3 — Smarter Budget + Tighter Accounting
Importance-weighted allocation (formally ε-DP by composition) + PRV accountant (51% tighter than RDP at k=100). More training, less noise, same privacy guarantee.
🌐
Validated — Both Findings are Universal
Cross-validated on NSL-KDD (2009) and UNSW-NB15 (2015). Same collapse, same recovery. MIA tests: 0/24 violations. 30-run Wilcoxon statistical tests throughout.
